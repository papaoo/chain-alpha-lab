# UI/UX 设计规范

## 1. 设计定位

风格：现代专业投研工具风。界面要像一个冷静、可信、带一点科技感的 A 股交易研究驾驶舱，而不是普通后台管理系统。

关键词：

- 清爽
- 克制
- 高信息密度
- 结论明确
- 数据依据醒目
- 风险提示清楚
- 现代感
- 轻量科技感
- 盘面氛围
- 决策路径清晰

避免：

- 花哨营销页。
- 只展示漂亮图表但没有结论。
- 多层卡片嵌套。
- 过度炫酷导致数字和结论难读。
- 把界面做成纯文本报告页。

视觉原则：

- 第一屏必须让用户立刻看到“今天能不能出手、主线是什么、风险在哪里”。
- 每个建议都要有“数据依据”入口，避免只给结论。
- 可加入动态行情氛围、抽象市场热力背景、主线流向视觉，但不能影响阅读。
- 科技感来自层级、光感、图表、动效和数据密度，不靠堆装饰。

## 2. 颜色规范

A 股习惯：

- 红色：上涨、正收益、净流入。
- 绿色：下跌、负收益、净流出。

辅助色：

- 蓝色：信息。
- 橙色：警告。
- 灰色：中性数据。

推荐主题：

- 默认深色主题：适合盯盘和夜间复盘，背景使用近黑/深灰，不使用大面积纯黑。
- 可选浅色主题：适合阅读历史报告和打印归档。
- 主色不做单一蓝紫渐变，避免廉价科技风。
- 红绿只用于交易语义，不用于普通装饰。

参考色值：

- 背景：`#0B0F14`、`#111827`
- 面板：`#151B23`、`#1F2937`
- 描边：`#2D3748`
- 正向红：`#EF4444`
- 负向绿：`#22C55E`
- 信息蓝：`#38BDF8`
- 警告橙：`#F59E0B`
- 文本主色：`#F8FAFC`
- 文本次色：`#94A3B8`

## 2.1 视觉资产规范

系统允许使用图片和动效增强专业感，但必须服务于投研场景。

可使用的图片资产：

- 抽象市场热力图背景：用于仪表盘顶部，表达资金流和板块温度。
- 主线雷达图或资金流向图：用于展示“市场 -> 板块 -> 个股”的分析路径。
- 空状态插画：用于无数据、接口失败、等待配置等场景。
- 风险提示视觉：用于防守行情、数据不足、模型输出被拦截等状态。

图片要求：

- 使用生成式图片时，必须是抽象投研/行情/数据流风格，不出现真实股票代码、虚构价格、虚构公司 Logo。
- 图片不得承载事实数据，事实数据只能来自后端事实包。
- 图片不能遮挡核心结论、价格、风险提示和操作建议。
- 关键界面不能依赖图片才可读，图片加载失败时仍能完整使用。

建议第一版生成 3 类素材：

- `market-heat-bg`：深色抽象市场热力背景，用于仪表盘顶部。
- `mainline-flow`：主线资金流向抽象图，用于主线页面。
- `risk-shield`：风险控制抽象图，用于防守/观望状态。

## 3. 页面结构

### 3.1 仪表盘

模块：

- 今日结论
- 大盘环境
- 当前主线
- 强股候选
- 公司认知卡片
- 风险提示
- 最近报告

第一屏布局：

- 顶部状态条：当前分析时段、数据时间、数据源、运行状态、重新分析按钮。
- 左侧主结论区：今日市场状态，显示“可交易 / 谨慎 / 防守”。
- 中间主线区：展示 1-3 条当前主线及阶段，如启动、确认、加速、分歧、退潮。
- 右侧风险区：显示最重要的 3 条风险和不买条件。
- 下方候选区：展示候选强股表格和买点状态。

第一屏必须避免长篇文字。长文本报告放在“展开详情”或“报告页”中。

时段展示要求：

- 第一屏必须显示 `盘前计划 / 集合竞价 / 早盘盯盘 / 午间复盘 / 午后确认 / 尾盘确认 / 收盘复盘 / 夜间研究 / 非交易日研究` 之一。
- 时段标签旁必须展示数据基准，例如“上一交易日收盘”“上午收盘快照”“盘中实时/延迟行情”“当日收盘数据”。
- 盘前、夜间和非交易日界面不得使用“正在盯盘”的视觉暗示；应显示计划、复盘或研究模式。
- 午间界面应突出“上午已验证 / 下午待验证”；尾盘界面应突出“收盘确认 / 隔日风险”。
- 时段信息应作为轻量状态条或提示卡，不得挤占主结论和风险约束。

### 3.2 今日大盘

展示：

- 上证、深成指、创业板、科创 50
- 分时判断文字
- 可交易/谨慎/防守状态
- 依据列表

### 3.3 市场主线

展示：

- 行业板块涨幅排名
- 概念板块涨幅排名
- 资金流入 Top
- 主线阶段判断

### 3.4 强股候选

每只股票展示：

- 股票名称和代码
- 当前角色：龙头/中军/补涨/观察
- 操作建议
- 买入条件
- 失效条件
- 数据依据

候选股推荐表字段：

- 代码/名称
- 所属主线
- 角色
- 趋势状态
- 资金状态
- 买点状态
- 建议动作
- 建议仓位上限
- 失效条件
- 数据完整性

视觉要求：

- 买点状态用图标和短标签表达，例如“等待回踩”“小仓试错”“不追”“回避”。
- 数据完整性必须可见，缺少 K 线、资金或板块证据时显示“数据不足”。
- 点击股票行后打开侧边详情，不跳转离开当前分析流。

### 3.4.1 公司认知卡片

每只推荐、观察或进入买卖计划的股票，都必须有公司认知卡片，帮助用户知道“这家公司到底是做什么的”，避免只按资金和技术指标交易。

展示内容：

- 公司名称、股票代码、所属行业。
- 核心业务一句话摘要。
- 主要产品或服务。
- 产业链位置：上游/中游/下游/终端应用。
- 与当前主线的匹配逻辑。
- 基本面看点。
- 基本面风险。
- 长线跟踪事项。
- 数据来源标签：数据源事实/规则计算/模型归纳。

交互要求：

- 候选股表格点击后，侧边详情第一屏展示公司认知卡片，再展示行情和买卖计划。
- “模型归纳”的内容必须有明显标签，避免用户误认为是公告事实。
- 如果公司信息不足，显示醒目的“公司信息不足，不支持长期逻辑判断”。

### 3.5 买卖计划

展示：

- 为什么买
- 何时买
- 错了怎么走
- 对了怎么看
- 仓位建议

### 3.6 配置中心

模块：

- DeepSeek 配置
- 通知配置
- 定时任务配置
- 策略参数

## 3.7 报告详情页

结构：

- 顶部：报告时间、数据源、市场状态、规则评分。
- 大盘判断：结论、依据、风险。
- 主线判断：板块、阶段、资金、持续性。
- 公司认知：核心业务、产业链、主题匹配和基本面风险。
- 个股计划：买入条件、卖出条件、仓位、失效条件。
- 原始依据：可折叠展示 westock-data 解析后的关键字段。
- 模型解释：DeepSeek 生成的自然语言解释。

要求：

- 规则结论和模型解释要分区显示，不能混在一起。
- 所有数字旁边需要能看到来源类型，如“行情数据”“规则计算”“模型解释”。
- 报告可复制、可导出 Markdown。

## 4. 组件规范

使用：

- 表格展示候选股。
- 标签展示主线阶段。
- 状态徽标展示可交易/谨慎/防守。
- 折叠面板展示数据依据。
- 提醒条展示风险。

推荐组件：

- MarketStateBadge：市场状态徽标。
- MainlineStrip：主线横条，展示板块强度、资金、阶段。
- StockSignalTable：候选股信号表。
- EvidenceDrawer：数据依据侧边抽屉。
- RiskBanner：风险提示条。
- PositionMeter：仓位上限仪表。
- DataCompletenessPill：数据完整性标签。
- TimelineReport：分析过程时间线。

动效要求：

- 页面切换和展开详情使用 150-220ms 轻动效。
- 市场状态变化可以使用轻微发光或边框变化。
- 不使用大面积闪烁、跳动、滚屏数字。
- 分析运行时展示清晰步骤：拉取数据、解析数据、规则判断、模型生成、校验结果。

## 5. 输出报告样式

报告必须先结论后依据，但每个结论必须有依据。

推荐结构：

```text
结论
数据依据
判断逻辑
建议
风险条件
```

报告视觉层级：

- 结论用短句，不超过两行。
- 数据依据用紧凑列表或小表格。
- 判断逻辑用 3-5 条要点。
- 风险和失效条件必须醒目，不能藏在正文末尾。

## 6. 移动端

第一版需保证移动端可读：

- 单列布局。
- 候选股卡片化展示。
- 表格支持横向滚动。

移动端补充要求：

- 顶部固定展示市场状态和“重新分析”按钮。
- 候选股卡片默认只展示结论、主线、买点和风险，依据可展开。
- 不在移动端展示复杂大图，图片资产只作为轻背景或空状态。

## 7. 信息架构

导航建议：

- 仪表盘
- 今日分析
- 主线板块
- 候选股票
- 模拟持仓
- 历史报告
- 预警通知
- 配置中心

## 8. MVP 界面优先级

第一阶段必须实现：

- 仪表盘。
- 今日分析报告。
- 候选股信号表。
- 证据详情抽屉。
- DeepSeek 配置。
- 通知配置。

第二阶段增强：

- 主题切换。
- 报告导出。
- 图片资产和空状态插画。
- 主线资金流向可视化。
- 历史报告对比。

## 9. 模拟持仓追踪界面（MVP 后第一增强）

### 9.1 模拟持仓列表

展示：

- 股票名称和代码。
- 模拟买入价。
- 近似最新价。
- 模拟盈亏。
- 所属主线。
- 计划周期。
- 当前追踪动作。
- 失效条件。
- 最新更新时间。
- 是否启用追踪。

视觉要求：

- 当前追踪动作必须醒目。
- “跌破失效条件”“主线退潮”“数据不足”等风险状态必须使用风险提示样式。
- “加仓”不得做成强刺激按钮，第一版只展示“等待加仓条件”或“满足加仓观察条件”。

### 9.2 创建模拟买入

入口：

- 候选股信号表行操作。
- 公司详情侧边栏。
- 模拟持仓页面新建按钮。

表单字段：

- 股票代码/名称。
- 模拟买入价格。
- 数量或仓位比例。
- 计划周期。
- 买入理由。
- 失效条件。
- 减仓条件。
- 继续持有条件。
- 是否启用追踪。

如果从候选股创建，表单应自动带入系统生成的买入逻辑、公司认知卡片和风控条件。

### 9.3 持仓详情

结构：

- 顶部：模拟盈亏、当前动作、最新追踪时间。
- 初始计划：买入理由、买点类型、失效条件、仓位约束。
- 逻辑检查：原买入逻辑是否仍成立。
- 当前数据：分时近似价格、日线趋势、技术指标、资金流、主线阶段。
- 公司认知：初始公司认知和最新补充。
- 追踪历史：每次追踪结论和触发依据。
- 通知记录：已发送提醒。

规则结论和模型解释仍需分区展示。
