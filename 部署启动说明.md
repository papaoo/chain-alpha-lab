# 部署启动说明

本文档说明 Chain Alpha Lab（链枢 Alpha）的本地开发、Windows 启动、Linux 部署和环境变量配置方式。

---

## 1. 基础环境

建议环境：

- Node.js 20+
- npm 10+
- Windows 10/11 或 Linux
- SQLite 本地数据库

安装依赖：

```bash
npm install
```

---

## 2. 环境变量

复制模板：

```bash
cp .env.example .env.local
```

Windows PowerShell：

```powershell
Copy-Item .env.example .env.local
```

常见配置：

```env
DATABASE_PROVIDER=sqlite
DATABASE_PATH=./data/app.db

MODEL_PROVIDER=deepseek
MODEL_BASE_URL=https://api.deepseek.com
MODEL_API_KEY=
MODEL_NAME=deepseek-v4-pro
MODEL_AUDIT_ENABLED=false

DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_API_KEY=
DEEPSEEK_MODEL=deepseek-v4-pro
```

注意：

- `.env.local` 不要提交到 GitHub。
- API Key 请只保存在本地或服务器环境变量中。
- 开源仓库只保留 `.env.example`。

---

## 3. 本地开发启动

```bash
npm run dev
```

默认访问：

```text
http://localhost:3000
```

如果需要指定端口，可以使用：

```bash
npx next dev -p 3006
```

---

## 4. Windows 启动

项目提供了 Windows 启停脚本：

```bat
start.bat
```

停止：

```bat
stop.bat
```

如果脚本端口和本机冲突，可以手动使用：

```powershell
npx next dev -p 3006
```

---

## 5. 生产构建

```bash
npm run build
npm run start
```

如果需要指定端口：

```bash
npx next start -p 3006
```

---

## 6. Linux 部署建议

### 6.1 安装依赖

```bash
git clone https://github.com/papaoo/chain-alpha-lab.git
cd chain-alpha-lab
npm install
cp .env.example .env.local
```

编辑 `.env.local`：

```bash
nano .env.local
```

### 6.2 构建

```bash
npm run build
```

### 6.3 启动

```bash
npx next start -p 3006
```

### 6.4 使用 pm2 托管

```bash
npm install -g pm2
pm2 start "npx next start -p 3006" --name chain-alpha-lab
pm2 save
```

查看日志：

```bash
pm2 logs chain-alpha-lab
```

重启：

```bash
pm2 restart chain-alpha-lab
```

停止：

```bash
pm2 stop chain-alpha-lab
```

## 7. 数据库说明

默认使用 SQLite：

```env
DATABASE_PATH=./data/app.db
```

数据库文件会在本地运行时生成。不要提交：

```text
data/app.db
data/app.db-shm
data/app.db-wal
```

建议定期备份：

```bash
npm run db:backup
```

查看数据库统计：

```bash
npm run db:stats
```

## 10. 常见问题

### 页面打不开

确认服务是否启动：

```bash
netstat -ano | findstr 3006
```

Linux：

```bash
lsof -i :3006
```

### 模型不可用

检查：

- `.env.local` 是否填写模型 API Key；
- 模型开关是否开启；
- baseUrl 是否正确；
- 网络是否能访问模型服务。

### 没有行情数据

检查：

- 东方财富公开接口是否可访问；
- westock-data skill 是否可用；
- Tushare token 是否配置；
- 当前是否非交易日；
- 数据源状态面板中的错误说明。

### 数据库过大

可以使用：

```bash
npm run db:archive
npm run db:backup
```

## 11. 安全提醒

不要提交以下内容：

- `.env.local`
- API Key
- webhook 地址
- SQLite 数据库
- 本地日志
- 个人过程文件

如果误提交了 key，请立即到对应平台重置。
