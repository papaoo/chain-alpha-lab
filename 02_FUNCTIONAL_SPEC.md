# 功能规格说明

## 1. 配置管理

### 1.1 DeepSeek 配置

字段：

- API Base URL
- API Key
- 模型名称
- 温度参数
- 超时时间
- 最大输出长度
- 是否启用大模型总结

默认建议：

- Base URL 可配置，不在代码中写死。
- 模型名可配置，例如 `deepseek-chat`、`deepseek-reasoner` 或用户自定义值。

### 1.2 通知配置

支持：

- 飞书 Webhook
- 企业微信 Webhook

字段：

- 渠道名称
- Webhook URL
- 是否启用
- 测试发送按钮

普通个人微信不直接支持官方机器人 Webhook。若需微信提醒，后续可通过企业微信、Server 酱、PushPlus 等方式间接推送。

### 1.3 扫描频率配置

默认任务：

- 09:45 早盘环境检查
- 10:30 主线确认
- 13:30 午后确认
- 14:30 尾盘买卖计划
- 15:10 收盘复盘

用户可启停任务。

## 2. 今日分析

按钮：

- 运行完整分析
- 只分析大盘
- 只分析板块
- 只筛选强股
- 测试通知

输出：

- 当前时间
- 数据源状态
- 规则引擎结论
- 大模型结论
- 大盘结论
- 主线结论
- 候选股票
- 买卖计划
- 风险提示

## 3. 大盘环境模块

数据来源：

- 指数分时：`minute sh000001`、`minute sz399001`、`minute sz399006`、`minute sh000688`
- 指数日线：`kline <index> --period day --limit 30`

输出：

- 可交易
- 谨慎交易
- 防守观望

说明：

- 该结论必须由程序规则引擎先生成。
- DeepSeek 可以解释该结论，但不得替代规则引擎自行重判为相反结果，除非明确标记为“模型补充观点”。

## 4. 主线识别模块

数据来源：

- `board`
- `hot board --limit 20`
- 板块 K 线：`kline <boardCode> --period day --limit 20`

输出：

- 主线板块名称
- 板块涨幅
- 主力净流入
- 5 日涨幅
- 20 日涨幅
- 阶段判断：启动、确认、加速、分歧、退潮

## 5. 强股筛选模块

数据来源：

- `hot stock --limit 50`
- `technical <codes> --group ma,macd,rsi`
- `asfund <codes>`
- `kline <codes> --period day --limit 30`
- `profile <codes>`
- 可选：`finance <codes> --num 4`、`shareholder <codes>`、`reserve <codes>`

过滤：

- ST 或风险警示股。
- 跌破 20 日线且 20 日线下行的股票。
- 明显远离 5 日线且处于高潮追涨状态的股票。
- 主力资金连续流出且无修复迹象的股票。

输出：

- 候选股票列表。
- 每只股票的角色：龙头、中军、补涨、低吸观察。
- 当前是否可买。
- 公司认知卡片。
- 买入条件。
- 风险条件。

说明：

- 候选池必须由后端程序生成。
- DeepSeek 只能在候选池内排序、解释和生成建议。
- DeepSeek 不得新增未经过数据模块校验的股票。

## 5.1 公司认知卡片模块

每只进入候选池、观察池或买卖计划的股票，都必须生成公司认知卡片，避免只基于数据指标买入。

数据来源：

- `profile <codes>`：公司名称、行业、主营业务、上市日期等基础信息。
- `finance <codes> --num 4`：最近多期财务摘要，若第一版解析复杂可作为增强项。
- `shareholder <codes>`：股东结构，作为增强项。
- `reserve <codes>`：业绩预告，作为增强项。
- 板块和主线数据：用于判断公司是否真实匹配当前市场主线。

输出字段：

- 公司名称和股票代码。
- 所属行业。
- 主营业务。
- 核心产品或服务。
- 产业链位置：上游/中游/下游/终端应用。
- 当前主线匹配逻辑。
- 基本面看点。
- 基本面风险。
- 长线关注点。
- 数据来源标记。

说明：

- 产业链解释可以由 DeepSeek 基于事实包归纳，但必须标记为“模型归纳”，不得伪装成数据源事实。
- 不得编造具体客户、订单、市占率、营收占比、利润增速等事实。
- 若公司基础信息缺失，系统不得给长期投资理由，只能提示“公司信息不足”。

## 6. 买卖计划模块

每只候选输出：

- 股票名称和代码
- 所属主线
- 公司认知摘要
- 数据依据
- 买入理由
- 买入条件
- 参考仓位
- 止损/失效条件
- 止盈/减仓条件
- 不买条件

## 7. 历史报告模块

保存：

- 分析时间
- 输入数据摘要
- 规则引擎结论
- 大模型输出
- 推送状态
- 错误日志

## 8. 提醒模块

触发场景：

- 定时报告。
- 主线变化。
- 候选股出现买点。
- 候选股失效。
- 大盘由可交易转谨慎或防守。

第一版只推送摘要，不做复杂交互。

## 9. 模拟持仓追踪模块（MVP 后第一增强）

该模块用于买后计划监督，不接券商账户，不自动交易。

### 9.1 创建模拟买入

入口：

- 候选股信号表中的“加入模拟追踪”。
- 模拟持仓页面手动新建。

字段：

- 股票代码和名称。
- 模拟买入价格。
- 模拟买入数量或仓位比例。
- 模拟买入时间。
- 买入理由。
- 所属主线。
- 买点类型。
- 计划周期：短线、波段、中线。
- 初始止损/失效条件。
- 初始减仓条件。
- 初始继续持有条件。
- 是否启用追踪。

说明：

- 如果从候选股创建，系统应默认带入当时的规则结论、公司认知卡片、买入条件和失效条件。
- 如果用户手动创建，但缺少 K 线、技术指标、资金流、板块证据或公司信息，系统只能标记为“数据待补充”。

### 9.2 定时追踪

数据来源：

- `minute <code>`：获取近似最新盘中价格和分时走势。
- `kline <code> --period day --limit 30`：判断趋势和关键均线位置。
- `technical <code> --group ma,macd,rsi`：判断均线、动能和过热风险。
- `asfund <code>`：判断主力资金流。
- `board`、`hot board`：判断所属主线和市场环境。
- `profile <code>`：补充公司基础信息。

限制：

- 当前版本 `quote` 不可用，不能依赖实时行情命令。
- `minute` 不支持批量，第一版应限制启用追踪的股票数量，或分批执行。

### 9.3 追踪输出

每次追踪输出：

- 当前模拟盈亏估算。
- 原买入逻辑是否仍成立。
- 大盘环境变化。
- 所属主线阶段变化。
- 个股趋势状态。
- 资金流状态。
- 公司认知状态是否变化。
- 建议动作。
- 触发依据。
- 风险和失效条件。

建议动作白名单：

- 继续观察。
- 继续持有。
- 风险升高，考虑减仓。
- 跌破失效条件，计划退出。
- 主线退潮，降低主题仓位。
- 等待加仓条件。
- 不追高加仓。
- 数据不足。

### 9.4 加仓约束

第一版不直接输出激进加仓建议，只能输出“满足加仓观察条件”或“等待加仓条件”。

加仓观察条件必须同时满足：

- 大盘不是防守状态。
- 所属主线未退潮。
- 原买入逻辑仍成立。
- 个股未高位严重远离均线。
- 资金没有连续明显流出。
- 当前模拟仓位未超过规则仓位上限。
- 出现新的明确买点，而不是单纯上涨。

### 9.5 通知触发

触发场景：

- 跌破用户或规则设置的失效条件。
- 主线由确认/加速转分歧或退潮。
- 大盘由可交易转谨慎或防守。
- 个股跌破 10 日或 20 日线。
- 主力资金连续流出。
- 达到减仓或退出条件。
- 模拟持仓数据不足，无法继续判断。
